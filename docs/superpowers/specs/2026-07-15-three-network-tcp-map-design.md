# 三网 TCP 延迟中国地图设计

## 目标

将 VPS 详情页的“三网 TCP 延迟”结果由表格改为交互式中国地图。地图颜色反映每个省份的三网平均 TCP 延迟；访客可查看运营商明细，不会产生新的测试或网络请求。

## 范围

- 保持现有 93 个 TCP 测试目标、任务批次、快照存储和管理员测试权限不变。
- 将当前结果表格替换为地图，保留测试时间、成功/失败统计、管理员开始/取消按钮及进行中的进度条。
- 中国地图包含 31 个已测试省级区域，以及香港、澳门、台湾三个固定无数据区域。
- 不修改 Komari 后端或 Agent，不加载第三方地图服务。

## 地图资源与加载

主题包内加入本地中国 GeoJSON，包含港澳台。允许主题包额外增加约 200 KB 至 1 MB。

`ThreeNetworkTcpLatency` 仍由详情页异步组件加载。GeoJSON 和 ECharts 地图模块只在用户打开“三网 TCP 延迟”标签后加载、注册和初始化。页面离开时销毁地图实例及事件监听；窗口尺寸变化通过节流的 `resize()` 更新。

GeoJSON 读取、地图注册或渲染失败时，组件显示“地图加载失败”提示；管理员测试、快照保存和既有记录均继续可用。

## 数据映射

新增独立的纯数据工具，将现有 `THREE_NETWORK_TARGETS` 顺序中的 93 项快照值映射为每个省份的数据项：

- 每省固定读取移动、联通、电信三项。
- 地图填充色采用有效数值的算术平均值。
- 一至两家失败时，仅以剩余有效值计算平均值。
- 三家均失败时，省份为灰色“无数据”。
- 香港、澳门、台湾始终为灰色“未测试”，不参与成功/失败统计和颜色图例。

地图名称与既有省份代码通过受控映射表匹配，避免 GeoJSON 名称与测试目录不一致而造成错误着色。

## 配色与图例

延迟色阶由省份平均值决定：

| 平均延迟 | 颜色 |
| --- | --- |
| 小于或等于 50 ms | 绿色 |
| 51 至 100 ms | 青绿色 |
| 101 至 180 ms | 黄色 |
| 181 至 300 ms | 橙色 |
| 大于 300 ms | 红色 |
| 无数据或未测试 | 灰色 |

地图内显示对应图例，使颜色含义无需依赖悬浮提示理解。

## 交互

桌面端悬浮省份显示提示卡，移动端轻触省份显示同一提示卡，轻触地图空白区域关闭。

提示卡显示省份名称、平均延迟和三家运营商明细，运营商顺序固定为：移动、联通、电信。每项显示毫秒值或“失败”。

## 性能与访客行为

访客只消费 `theme_settings` 中已有的快照。地图展示不调用管理员 API、RPC 或轮询接口；没有快照时展示现有的“管理员尚未完成测试”空状态。

地图只维护一个 ECharts 实例。无数据状态不会创建额外图表或后台定时器。

## 测试

- 93 项快照按目标目录映射到 31 个省份。
- 验证移动、联通、电信的读取顺序，以及可用值平均规则。
- 验证部分失败、全部失败、港澳台无数据与五档阈值边界。
- 验证地图组件懒加载、本地 GeoJSON 资源和失败提示契约。
- 保留现有快照版本 `1`，旧记录无需迁移且可直接在地图中展示。
