# Live2D 资源主题路由兼容设计

## 概要

部分已部署的 Komari 版本使用单数静态资源路由 `/theme/:id/*path`，当前版本使用复数路由 `/themes/:id/*path`。Naive Extended 目前只允许复数路径，导致旧版 Komari 中实际可访问的 Live2D 模型入口被主题设置规范化为错误的默认地址，看板娘无法加载。

本修复同时接受单数和复数固定资源包路径，让管理员填写其 Komari 实际可访问的模型入口。修复不修改 Komari 后端、不自动探测版本，也不增加失败重试或额外网络请求。

## 目标

- 兼容使用 `/theme/` 和 `/themes/` 静态路由的 Komari 部署。
- 保留独立资源包 `komari-live2d-models` 和现有模型目录约束。
- 保持当前安全校验，不允许借兼容逻辑扩大到任意主题或外部 URL。
- 不增加访客 CPU、RAM、网络请求或宿主机负载。

## 非目标

- 不修改 Komari 后端或检测 Komari 版本。
- 不在请求失败后自动切换另一个路径。
- 不迁移、复制或重命名管理员的模型文件。
- 不改变 Live2D 渲染、缩放、指针跟随、欢迎语或会话关闭逻辑。

## 路径契约

允许的模型入口前缀为：

```text
/themes/komari-live2d-models/dist/model/
/theme/komari-live2d-models/dist/model/
```

默认入口继续使用当前 Komari 的复数路由：

```text
/themes/komari-live2d-models/dist/model/model.model3.json
```

管理员必须填写其部署中实际可访问的路径。旧版部署示例：

```text
/theme/komari-live2d-models/dist/model/XFZN.model3.json
```

路径仍必须满足以下全部条件：

- 使用站内绝对路径，且属于上述两个固定前缀之一。
- 以 `.model3.json` 结尾。
- 不包含查询参数、片段、用户名、密码或协议前缀。
- 不包含反斜杠、编码后的反斜杠或 `..` 路径穿越。
- 不允许访问其他主题、主主题、管理接口或外部域名。

模型 JSON 内部的 Moc、纹理、物理、动作、表情和声音引用继续相对于入口文件解析，并且必须留在入口所在目录或其子目录中。

## 实现边界

`src/utils/live2dCompanion.ts` 将固定前缀从单个常量扩展为只读前缀集合。路径验证函数只需判断解析后的 pathname 是否匹配其中一个前缀；后续模型 URL 解析和模型内部引用验证保持不变。

`komari-theme.json` 的字段类型和默认值不变，只更新帮助文字，明确支持 `/theme/` 与 `/themes/`。README 和资源包内说明增加新旧路径示例，避免管理员继续被默认路径误导。

不实现自动回退。自动回退会让旧版部署每次首次加载产生一次无意义的 404，并使错误诊断、缓存和安全审计更复杂；允许管理员直接填写真实路径更轻量且可预测。

## 失败处理

- 不属于两个固定前缀的路径继续规范化为复数默认入口。
- 合法但服务器不存在的入口按现有行为安静隐藏看板娘并输出一次控制台警告。
- 模型资源损坏、WebGL 不可用和渲染初始化失败的处理保持不变。
- 路径兼容修复不得影响探针主体页面。

## 测试策略

- 单元测试确认单数与复数根目录、嵌套目录和非 ASCII 文件名均可解析。
- 单元测试继续拒绝其他主题、外部 URL、查询参数、片段、反斜杠、路径穿越和非 `.model3.json` 文件。
- 设置测试确认单数路径不会被重置为默认值。
- 契约测试确认默认值仍为复数路径，并且帮助文字说明两种路由。
- 文档测试确认 README 与资源包说明包含旧版单数路径示例。
- 最终运行 `pnpm test:unit`、`pnpm lint` 和 `pnpm build`。

## 验收标准

- 在单数路由部署中填写 `/theme/komari-live2d-models/dist/model/XFZN.model3.json` 后，看板娘使用该地址加载。
- 在复数路由部署中现有 `/themes/...` 设置不回归。
- 不产生探测请求、404 回退请求或服务端改动。
- 原有路径安全边界、性能策略和 Live2D 交互行为保持不变。
