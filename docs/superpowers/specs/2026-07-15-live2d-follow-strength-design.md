# Live2D 鼠标跟随幅度设置设计

## 概要

为 Live2D 看板娘新增一个管理员后台数字设置，用百分比统一控制鼠标与触摸驱动的注视、头部和身体跟随幅度。该设置只影响传给 Cubism focus controller 的归一化坐标，不改变看板娘容器、模型缩放、底部锚点、帧率、事件监听或网络行为。

## 目标

- 管理员可在主题后台将鼠标跟随强度调整为 `0–200%`。
- `100%` 精确保留当前横向 `0.35`、纵向 `0.22` 的效果。
- `0%` 停止传递偏移目标，模型保持回正，同时保留显示、欢迎语、点击和关闭交互。
- 允许在不增加访客 CPU、内存、网络或宿主机负担的情况下实时应用设置。

## 非目标

- 不拆分横向和纵向为两个后台设置。
- 不新增跟随强度预设、滑块、动画循环、计时器或请求。
- 不改变模型位置、模型缩放、Canvas 尺寸、已有 FPS 档位或模型资源加载流程。
- 不修改 Komari 后端、数据库或主题设置存储方式。

## 设置契约

新增 managed setting：

```json
{
  "key": "live2dFollowStrength",
  "name": "Live2D 鼠标跟随幅度",
  "type": "number",
  "default": 100,
  "help": "鼠标与触摸跟随强度百分比，0-200；100 为默认效果，0 为关闭跟随"
}
```

主题 Store 只接受有限数字；缺失、字符串、`NaN`、无穷大或其他非法值统一使用 `100`。运行时再次将数值限制在 `0–200`，避免主题设置来源变化绕过 Store 时导致异常坐标。

## 坐标与运行时行为

现有页面坐标先映射至 `[-1, 1]`，再以横向 `0.35`、纵向 `0.22` 形成注视目标。新增强度系数：

```text
factor = clamp(live2dFollowStrength, 0, 200) / 100
x = normalizedX * 0.35 * factor
y = normalizedY * 0.22 * factor
```

因此：

| 设置 | 横向最大值 | 纵向最大值 |
| --- | ---: | ---: |
| `0` | `0` | `0` |
| `100` | `0.35` | `0.22` |
| `200` | `0.70` | `0.44` |

组件沿用既有被动指针监听和下一渲染帧应用机制。它仅把 Store 的强度传入坐标工具函数；不添加 RAF、watcher 以外的生命周期资源、DOM 写入或 Pixi 调用。强度变化会加入现有运行时重建 watch 列表，以便后续指针事件立即使用新配置；不会重新请求模型、Core 或 IP 服务。

`0%` 仍保留指针事件的最低限度路径，但所有合法输入解析为 `{ x: 0, y: 0 }`，由既有 Cubism focus controller 完成回正。这样不影响触摸结束、失焦、隐藏页面和销毁时的既有清理语义。

## 安全与失败处理

- 强度是纯本地数值，不改变模型路径、请求 URL 或资源引用验证。
- 非法主题配置不抛出错误、不显示弹窗，并回退到 `100%`。
- 非有限页面坐标或零尺寸视口继续返回 `null`。
- 已销毁、隐藏、reduced-motion 或无 WebGL 时，既有无操作与静态帧行为保持不变。

## 测试策略

- 坐标工具函数测试 `0%`、`100%`、`200%`、越界值和非法强度。
- 设置测试覆盖清单默认值、帮助文字、合法数值以及字符串、无穷大和越界值的归一化。
- 组件契约测试确认现有指针事件将 Store 强度传入坐标工具函数，并把该设置加入运行时重建 watch。
- 全量测试、类型检查、lint 和生产构建均需通过。

## 验收标准

- 管理员设置 `0`、`100`、`200` 时，最大目标分别为 `0/0`、`0.35/0.22`、`0.70/0.44`。
- 现有 `live2dScale` 只控制展示尺寸；新设置只控制内部跟随目标，两者互不影响。
- 不新增客户端常驻任务或服务端工作。
- 看板娘显示、模型加载、欢迎语、点击、关闭、缩放、路由兼容和安全校验无回归。
