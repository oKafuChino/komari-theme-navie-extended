# Live2D 模型与主题分离设计

## 目标

让 Live2D 运行时随主题更新，而管理员模型保存在主题 ZIP 之外。主题只加载管理员在设置中填写的模型入口 URL，不修改 Komari 后端、不保存模型或访客 IP。

## 方案

`live2dModelPath` 从“主题内 `/live2d/` 路径”扩展为模型入口 URL。允许相对路径（默认 `/live2d-model/model.model3.json`）和 `http`/`https` 绝对 URL。模型入口必须以 `.model3.json` 结尾、无查询参数/片段/认证信息；模型入口及其 `FileReferences` 必须保持同源并位于入口目录内，避免远程资源注入和路径穿越。跨源入口因此会被拒绝，管理员可使用同源持久静态目录；保留外部 URL 配置字段仅为未来/部署兼容，不绕过资源安全边界。

主题包继续包含 `/live2d/runtime/` 的 Cubism Core 与渲染依赖说明，但不包含 `public/live2d/model/` 模型目录。更新主题 ZIP 不会触碰管理员单独部署的 `/live2d-model/` 目录。

## 配置与兼容

- 默认入口改为 `/live2d-model/model.model3.json`。
- 旧的 `/live2d/model/...` 仍可被解析，以避免已有部署立即失效；文档标记为旧路径，建议迁移到主题外目录。
- 无效、跨源或非 `.model3.json` 地址会隐藏看板娘并记录一次警告。
- Live2D 禁用时继续不请求模型、运行时或 IP 服务。

## 更新流程

管理员首次部署时把完整模型目录放入 Komari 实例可持久化的同源静态目录（例如 `/live2d-model/`），在主题设置填写入口。后续从管理页面更新主题只替换主题资源，模型目录和设置值保持不变；若部署环境没有主题外静态目录，则需由管理员继续维护外部同源静态服务器。

## 验证

单元测试覆盖默认设置、旧路径兼容、同源持久路径、跨源拒绝、模型引用边界和发布 ZIP 不含模型文件。生产构建、lint 与现有 Live2D/环境特效测试必须继续通过。
